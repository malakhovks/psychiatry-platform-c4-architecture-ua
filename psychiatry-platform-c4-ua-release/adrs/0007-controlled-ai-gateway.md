# 7. Усі моделі ШІ викликаються через контрольований оркестратор

Дата: 2026-06-20

## Статус

Прийнято

## Контекст

Прямі виклики моделей із клієнтів або доменних сервісів унеможливлюють єдині політики приватності, версійності та безпеки.

## Рішення

Використати єдиний AI Orchestrator з model routing, мінімізацією даних, prompt registry, RAG, pre/post guardrails, schema validation, human-control і повним походженням.

## Наслідки

Оркестратор стає критичною точкою керування й потребує високої доступності для допоміжних функцій, але його відмова не зупиняє детерміноване ядро.
