# Архітектура програмної платформи психіатричних сценаріїв та алгоритмів

Цей пакет містить повну C4-модель універсальної програмної платформи для формалізації, версіонування, виконання, клінічного контролю та аудиту сценаріїв у сфері психіатрії. Усі назви, описи елементів, зв'язків і діаграм подано українською мовою.

## Оновлені слайдові C1-подання

Діаграми `C1-01-Landscape`, `C1-03-PatientContext` і `C1-04-ClinicianContext` оптимізовано для презентаційного формату 16:9 та експортовано у роздільності 1920×1080. У C1.01 вилучено ролі «Черговий кризовий фахівець» і «Клінічний методолог», а також додано сині ролі «Кардіолог», «Реабілітолог», «Невролог» і «Педіатр». C1.03 ущільнено навколо взаємодії пацієнта і лікаря. До C1.04 додано чотири зазначені профільні лікарські ролі синього кольору. Для всіх трьох діаграм застосовано метричне припасування тексту: кожний напис перевіряється за шириною та висотою і залишається всередині відповідної фігури або текстової області. Відтворюваний генератор розміщено у `scripts/render_c1_presentation.py`, а готову презентацію — у `C1_updated_slide_diagrams_UA.pptx`.

## Склад результату

- `workspace.dsl` — головний модульний Structurizr DSL workspace;
- `workspace-single-file.dsl` — самодостатня однофайлова версія для перенесення та перевірки;
- `model/` — актори, зовнішні системи, контейнери, компоненти, контракти й вузли розгортання;
- `views/` — C1, C2, C3, L4, динамічні та deployment-подання;
- `docs/04-diagram-catalog.md` — повний текстовий опис усіх 41 діаграм;
- `docs/03-universal-workflow.md` — універсальний процес виконання сценарію;
- `schemas/algorithm-definition.schema.json` — формальний контракт визначення алгоритму;
- `examples/` — шаблон алгоритму, каталог профілів і шаблон контрольного тесту;
- `adrs/` — 10 записів про ключові архітектурні рішення;
- `previews/` — SVG, PNG, DOT і HTML-індекс попереднього перегляду;
- `scripts/` — складання, статична перевірка, створення попередніх переглядів, контроль припасування тексту, PPTX і документації;
- `sources/high-level-schema.jpg` — надана високорівнева схема.

## Ключова архітектурна межа

Обчислення балів, підшкал, порогів, умов переходу, червоних прапорців, кризових правил і маршрутизації виконує детермінований версійований рушій. Генеративний ШІ не змінює ці правила і використовується тільки для допоміжної комунікації, транскрипції, структурованого вилучення, пошуку у перевіреній базі знань та формування проєктів тексту. Клінічно значущий результат потребує контролю уповноваженого фахівця.

## Універсальність алгоритмів

Специфікація підтримує профілі PHQ-9, GAD-7, PQ-16, MDQ, CFQ, MoCA, AUDIT-C, DAST, оцінювання психічного статусу, кризового реагування, ЕКГ за Міннесотським кодом, голосового та відеоаналізу. Конкретні тексти запитань, пороги, локальні правила та права використання не дублюються в архітектурі: їх слід імпортувати з погодженого репозиторію джерел, верифікувати клінічно та опублікувати як окремі незмінні версії.

## Перегляд у Structurizr

У каталозі пакета виконайте:

```bash
docker pull structurizr/structurizr
docker run -it --rm -p 8080:8080 \
  -v "$PWD":/usr/local/structurizr \
  structurizr/structurizr local
```

Після запуску відкрийте `http://localhost:8080`. Structurizr використовує `workspace.dsl`, який підключає всі модульні файли через `!include`.

## Перевірка

Локальна статична перевірка без зовнішніх залежностей:

```bash
./scripts/validate.sh --static-only
```

Повна офіційна перевірка Structurizr через Docker:

```bash
./scripts/validate.sh
```

Сценарій спочатку формує `workspace-single-file.dsl`, перевіряє унікальність ключів, посилання, відношення і структуру подань, валідність JSON/YAML, наявність усіх графічних подань та метричне розміщення 75 текстових блоків у C1.01, C1.03 і C1.04. За наявності Docker додатково запускається офіційна команда Structurizr `validate`.

## Експорт

Попередні SVG/PNG вже містяться в `previews/`. Для повторної побудови локальних Graphviz-переглядів:

```bash
./scripts/export.sh graphviz
```

Для офіційного експорту Structurizr у Mermaid, C4-PlantUML або статичний HTML:

```bash
./scripts/export.sh mermaid
./scripts/export.sh c4plantuml
./scripts/export.sh static
```

## Трасування до вихідних матеріалів

Надана високорівнева схема використана для моделювання двох головних контурів: ШІ-асистента пацієнта та ШІ-асистента медичного працівника. Папка Google Drive «Команда Психіатрії алгоритм» розглядається як авторитетний репозиторій клінічного змісту. Архітектура навмисно відокремлює структуру сценарію від конкретного тексту інструменту, щоб однаковий workflow застосовувався до всіх алгоритмів, а клінічні зміни проходили контрольоване версіонування й погодження.

Деталі наведено у `docs/00-source-traceability.md` та `docs/04-diagram-catalog.md`.
